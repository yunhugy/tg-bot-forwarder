### 方案B: 手动 GitHub + Vercel

1. **推送到 GitHub**
   ```bash
   cd /var/minis/workspace/tg-bot-forwarder
   git init
   git add .
   git commit -m "Add Telegram bot"
   git remote add origin <你的仓库地址>
   git push -u origin main
   ```

2. **Vercel 部署**
   - 访问 https://vercel.com
   - Import Repository → 选择项目
   - 环境变量：
     ```
     TG_BOT_TOKEN = 你的BotToken
     ADMIN_ID = 6609386680
     ECHO_MODE = true
     ```
   - Deploy

3. **设置 Webhook**
   ```bash
   curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://<项目域名>.vercel.app/"
   ```

## 📡 Webhook 设置

部署后执行：

```bash
# 获取域名
echo "你的 Vercel 域名: https://<项目名>.vercel.app"

# 设置 Webhook
curl -X POST "https://api.telegram.org/bot$TG_BOT_TOKEN/setWebhook?url=https://<域名>.vercel.app/"
```

验证 Webhook:
```bash
curl "https://api.telegram.org/bot$TG_BOT_TOKEN/getWebhookInfo"
```

## 🤖 使用教程

### 用户端 (匿名用户)
1. 搜索你的 Bot (用户名: @你的Bot)
2. 发送消息 (文字/图片/文件)
3. 收到回复

### 管理员端 (你)
1. **接收消息**: 用户消息 → Bot → 你
2. **回复消息**: 
   - 点击「↩️ 回复」按钮
   - 或直接回复消息
   - 或发送 `/reply_<用户ID> 内容`
3. **管理对话**: 发送 `/list` 查看所有用户

### 命令速查
| 命令 | 功能 |
|------|------|
| `/list` | 查看所有活跃对话 |
| `/reply_123456 你好` | 回复指定用户 |
| (点击回复按钮) | 快速回复 |

## 🐛 故障排除

### Q1: Webhook 设置失败
```bash
# 检查 Bot Token
python3 test_bot.py

# 检查域名是否可访问
curl -I "https://<域名>.vercel.app"

# 重新设置 Webhook
curl -X POST "https://api.telegram.org/bot$TG_BOT_TOKEN/setWebhook?url=https://<域名>.vercel.app/"
```

### Q2: Bot 不转发消息
- 检查 Vercel 函数日志
- 确保环境变量正确
- 检查用户是否被 Bot 屏蔽

### Q3: 部署后域名无法访问
- 等待几分钟 (DNS 传播)
- 检查 Vercel 部署状态
- 确认环境变量已设置

## 🔒 安全性建议

1. **保护 Bot Token**：不要分享
2. **用户隐私**：Bot 会显示用户 ID，不显示其他敏感信息
3. **防滥用**：对话数据在内存中，重启 Vercel 会清空

## 🆘 支持

1. **重置 Bot**: 删除 @BotFather 重设 Token
2. **重新部署**: Vercel → Project → Redeploy
3. **检查日志**: Vercel → Functions → Logs
